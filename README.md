# ربات کانفیگ رایگان V2Ray (FreeV2Ray Bot)

رباتی برای تلگرام که به کاربران کانفیگ رایگان V2Ray می‌دهد؛ ساخته‌شده با **Laravel 13 + MySQL + Redis**، اجرا روی **Docker**، با یک **پنل مدیریت وب (Filament)** و مدیریت سریع داخل خود ربات.

## ✨ امکانات

**سمت کاربر (ساده):**
- 🎁 دریافت کانفیگ — کاربر می‌تواند کانفیگ جدید بگیرد یا کانفیگ فعلی‌اش را تمدید کند.
- 📚 آموزش‌ها — راهنمای اتصال (متن/عکس/ویدیو) که ادمین مدیریت می‌کند.
- 👥 زیرمجموعه‌گیری — لینک دعوت، تعداد زیرمجموعه‌ها و هدیه‌ها.

- 👤 پروفایل — آیدی عددی، اشتراک فعال و تاریخچه‌ی کانفیگ‌ها.

**سمت ادمین:**
- افزودن چند **پنل** دلخواه با پشتیبانی از **3x-ui (سنایی)**، **PasarGuard** و **Remnawave**.
- تعیین **حجم و مدت زمان** کانفیگ از طریق «پلن‌ها».
- **سیستم رفرال** قابل فعال/غیرفعال‌سازی: «به ازای هر N زیرمجموعه → X حجم یا روز هدیه» (قانون تکرارشونده یا پلکانی).
- **قفل عضویت اجباری کانال** کامل: ادمین یک پیام از کانال (عمومی/خصوصی) فوروارد می‌کند، ربات آیدی را برمی‌دارد، لینک دعوت اختصاصی می‌سازد و آمار ورود از آن لینک را ثبت می‌کند.
- **متن‌ها و دکمه‌ها در دیتابیس** و قابل ویرایش توسط ادمین؛ پشتیبانی از **ایموجی پریمیوم** (در متن‌ها با `<tg-emoji>` و روی دکمه‌ها با `icon_custom_emoji_id` — نیازمند Premium بودنِ مالک ربات).
- **ضد اسپم و بلاک موقت**، **روشن/خاموش‌کردن کل ربات**، **بلاک کاربر با آیدی عددی**.
- **نحوه‌ی تحویل** قابل تنظیم: لینک ساب یا کانفیگ‌های تکی.
- **گزارش‌دهی به گروه تاپیک‌بندی‌شده** (هر رویداد در تاپیک خودش): کاربر جدید، کانفیگ، رفرال، خطا، بلاک.
- **پیام همگانی**، آمار و گزارش‌ها، حالت تعمیر — همه از پنل مدیریت داخل ربات و پنل وب.

## 🏗 معماری

| لایه | فناوری |
|---|---|
| فریم‌ورک | Laravel 13 (PHP 8.3) |
| ربات | [Nutgram](https://nutgram.dev) (وب‌هوک) |
| پنل وب | Filament v5 (`/admin`) |
| دیتابیس | MySQL 8 |
| صف/کش | Redis (Predis) |
| وب‌سرور | FrankenPHP (با HTTPS خودکار) |

پنل‌ها از طریق یک رابط واحد `App\Panels\Contracts\PanelDriver` و درایورهای جداگانه مدیریت می‌شوند. همه‌ی فراخوانی‌های پنل در **صف (queue job)** انجام می‌شود تا وب‌هوک سریع پاسخ دهد.

```
app/
├─ Panels/            رابط درایور پنل + ۳ درایور + DTOها
├─ Services/          منطق دامنه (صدور کانفیگ، رفرال، قفل کانال، انتخاب پنل)
├─ Telegram/          هندلرها، میدلورها، کیبوردها، گفتگوها (Nutgram)
├─ Jobs/              صدور کانفیگ، پیام همگانی، اعمال هدیه رفرال
├─ Filament/          منابع، ویجت‌های داشبورد، صفحه‌ی تنظیمات
└─ Console/Commands/  انقضا، همگام‌سازی مصرف، بررسی سلامت پنل
```

## 🚀 نصب سریع (اینستالر)

ساده‌ترین راه. روی سرور لینوکسی که Docker دارد:

```bash
git clone <repo> freev2ray && cd freev2ray
chmod +x install.sh
./install.sh
```

اینستالر دامنه، توکن ربات و آیدی عددی ادمین را می‌پرسد، تمام کلیدها/رمزها را می‌سازد، سرویس‌ها را بالا می‌آورد و وب‌هوک را ثبت می‌کند. (تمام پیام‌های اینستالر انگلیسی است.)

> پیش‌نیاز: رکورد DNS دامنه به IP سرور اشاره کند و پورت‌های ۸۰ و ۴۴۳ باز باشند (Caddy گواهی TLS را خودکار می‌گیرد).

## 🚀 راه‌اندازی دستی با Docker

پیش‌نیاز: `docker` و `docker compose`.

```bash
# ۱) فایل محیط
cp .env.example .env

# ۲) مقادیر مهم را در .env تنظیم کنید:
#    SERVER_NAME=bot.example.com          (دامنه‌ی شما؛ برای HTTPS خودکار)
#    APP_URL=https://bot.example.com
#    TELEGRAM_TOKEN=...                    (از @BotFather)
#    TELEGRAM_BOT_USERNAME=...             (بدون @)
#    TELEGRAM_ADMIN_IDS=123456789          (آیدی عددی ادمین‌ها، با کاما)
#    TELEGRAM_WEBHOOK_SECRET=...           (یک رشته‌ی تصادفی طولانی)
#    ADMIN_EMAIL / ADMIN_PASSWORD          (ورود به پنل وب)
#    DB_PASSWORD / DB_ROOT_PASSWORD        (دلخواه)

# ۳) ساخت کلید اپلیکیشن و قرار دادن آن در .env (APP_KEY=...)
docker compose run --rm init php artisan key:generate --show
#  خروجی را در APP_KEY بگذارید، سپس:

# ۴) بالا آوردن سرویس‌ها (مهاجرت‌ها و seed خودکار اجرا می‌شوند)
docker compose up -d --build

# ۵) ثبت وب‌هوک تلگرام (allowed_updates شامل chat_member برای آمار کانال)
docker compose exec web php artisan bot:set-webhook
```

اکنون:
- ربات تلگرام فعال است.
- پنل وب: `https://bot.example.com/admin` (با ایمیل/رمز ادمین).

> امنیت وب‌هوک: علاوه بر بخش مخفی مسیر (`/webhook/<secret>`)، در حالت production نوتگرام هدر `X-Telegram-Bot-Api-Secret-Token` را نیز (برابر `md5(APP_KEY)`) بررسی می‌کند که `nutgram:hook:set` آن را خودکار تنظیم می‌کند.

## ⚙️ پیکربندی پس از نصب (در پنل وب)

1. **سرویس‌ها → پنل‌ها:** یک پنل اضافه کنید. فیلدهای اختصاصی بر اساس نوع:
   - **3x-ui:** `base_url`، `username/password` یا `api_token`، `inbound_id`، و `sub_host/sub_port/sub_scheme/sub_path` برای ساخت لینک ساب. (لینک ساب توسط ربات ساخته می‌شود چون API این پنل آن را برنمی‌گرداند.)
   - **PasarGuard:** `base_url`، `username/password`، و در صورت نیاز `group_ids` و `proxy_settings`.
   - **Remnawave:** `base_url`، `api_token`، و `squad_uuids`.
   با دکمه‌ی **«تست اتصال»** صحت تنظیمات را بررسی کنید.
2. **سرویس‌ها → پلن‌ها:** حجم (بایت) و مدت (روز) کانفیگ پیش‌فرض را تعیین کنید (`0` = نامحدود).
3. **رفرال → قوانین رفرال:** قانون «هر N نفر → هدیه» را بسازید/فعال کنید.
4. **محتوا و قفل → کانال‌های اجباری / آموزش‌ها** را مدیریت کنید.
5. **تنظیمات ربات:** روشن/خاموش کردن قفل کانال و رفرال، پیام خوش‌آمد، یوزرنیم ربات و... .

## 🧪 توسعه‌ی محلی و تست

```bash
composer install
php artisan migrate --seed       # نیازمند افزونه‌ی pdo_sqlite یا اتصال MySQL
php artisan test                 # مجموعه‌ی تست (درایورها، رفرال، جریان ربات)
```

برای توسعه می‌توانید به‌جای وب‌هوک از long-polling استفاده کنید:

```bash
php artisan nutgram:run
```

## 🛠 کارهای زمان‌بندی‌شده

سرویس `scheduler` به‌صورت خودکار اجرا می‌کند:
- `configs:expire` — منقضی‌کردن کانفیگ‌های تمام‌شده و غیرفعال‌سازی روی پنل.
- `configs:sync-usage` — همگام‌سازی مصرف/انقضا از پنل‌ها.
- `panels:health-check` — بررسی سلامت پنل‌ها.
